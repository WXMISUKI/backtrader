## 1. 背景

Phase 53 到 Phase 58 已经把诊断语义、摘要模板、证据视图、回看对齐、验收收口包都做出来了。

但这些能力分散在多个脚本和文档里，虽然都能用，但对日常维护来说还不够“一眼可懂”。

所以本阶段不再新增能力，而是把这条诊断链路整理成一份清晰的路径说明，帮助后续快速定位：

- 从哪里开始看
- 哪个脚本负责什么
- 哪些材料是统一消费的

---

## 2. 目标

### 2.1 总目标

把 53 到 58 的诊断链路整理成一份统一路径说明，方便日常和后续维护快速阅读。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 路径清晰 | 明确每个入口的职责 |
| 顺序明确 | 明确推荐的查看顺序 |
| 材料统一 | 说明哪些材料是共享的 |
| 便于回溯 | 后续回看时不需要翻多份文档 |

---

## 4. 设计原则

### 4.1 只整理，不发明

路径说明只总结现有成果，不新增业务语义。

### 4.2 从使用角度出发

优先回答“我现在该看哪个入口”和“这份材料归谁管”。

### 4.3 保持轻量

路径说明应短而清晰，不写成长篇设计回顾。
