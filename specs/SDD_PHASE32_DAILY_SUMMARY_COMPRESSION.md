## 1. 背景

Phase 31 已经把自选股日常流程统一起来，能一次跑完：

- 数据健康预检
- 持仓上下文
- 日常决策清单

但统一入口的输出仍然偏长，适合留档，不够适合每天第一眼快速扫读。日常使用里最常见的需求不是看全量明细，而是快速抓住：

- 今天最需要关注什么
- 哪些标的有变化
- 是否有明显风险或数据降级
- 持仓里有没有需要特别留意的标的

因此，本阶段不再增加新流程，而是给统一入口加一层“日更摘要压缩”，把最重要的信息收敛到更短的首屏输出。

---

## 2. 目标

### 2.1 总目标

让统一入口在保留完整输出的同时，额外生成一个简短的日更摘要，降低每天查看成本。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 首屏更短 | 让用户先看到当天最重要的几条信息 |
| 保留原文 | 不删除原始的健康预检和决策明细 |
| 只做压缩 | 不新增新的判断逻辑 |
| 便于分享 | 摘要更适合复制、截图或留档 |
| 兼容现有入口 | 不破坏 Phase 31 的统一流程 |

## 4. 设计原则

### 4.1 先摘要，后明细

先输出一段最短摘要，再输出完整结构，方便快速浏览。

### 4.2 摘要只负责压缩

不重新发明规则，只把已有的健康、仓位和决策结果压缩成更短的读法。

### 4.3 保持可追溯

摘要必须能回指到完整输出，不做黑箱总结。

### 4.4 兼容日常习惯

用户仍可继续看完整输出和 JSON，摘要只是首屏增强。

---

## 5. 输入输出约定

### 5.1 输入

统一入口现有输入继续沿用：

- `--watchlist`
- `--portfolio`
- `--start-date`
- `--end-date`
- `--risk-profile`
- `--limit`
- `--output-json`

### 5.2 输出

新增一个首屏摘要块，建议包含：

- 今日总体判断
- 数据健康状态
- 重点关注标的
- 继续观察标的
- 已持有且需留意的标的
- 需回避或暂不行动的标的

JSON 中建议增加：

- `daily_summary`
- `daily_highlights`

---

## 6. 验收标准

- 统一入口会先输出更短的摘要
- 摘要能概括健康、持仓和决策的核心变化
- 完整明细仍然保留
- JSON 中保留摘要和明细
- 不影响 Phase 31 统一入口的执行顺序
