## 1. 背景

项目当前已经具备东方财富行情、akshare 基本面、回测、分析和智能体接入能力。但真实数据链路仍然存在一个典型问题：上层能力已经可以调用，底层数据却可能因网络、接口变更、限流或返回结构变化而失败。

如果失败只表现为异常抛出或临时打印日志，那么智能体会拿到不稳定输出，回测和分析也难以追溯真实原因。

因此需要把真实数据失败这件事，变成一个可识别、可降级、可审计、可被智能体理解的标准流程。

---

## 2. 目标

### 2.1 总目标

建立一条统一的数据治理路径，让行情、基本面和回测入口都能在真实数据不可用时稳定返回受控结果，并显式标记降级状态。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 降级显式化 | 真实失败、缓存命中、模拟回退必须可见 |
| 来源标准化 | 统一使用 `real / mock / cache / config / unknown` |
| 结果可追踪 | 返回质量、原因、时间戳和上下文 |
| 接口兼容 | 保留现有函数签名，避免打断已有调用 |
| 智能体友好 | 输出结构适合工具调用和自然语言总结 |

---

## 3. 范围

### 3.1 纳入范围

- 东方财富行情获取
- akshare 基本面获取
- 回测数据获取
- 数据快照与质量检查
- 统一的降级标记与原因说明

### 3.2 不纳入范围

- 分布式缓存
- 数据仓库
- 自动补数
- 自动修复接口解析逻辑

---

## 4. 设计原则

### 4.1 先兼容，再增强

所有旧入口保留原有返回类型，新能力通过额外的 governed 入口或可选参数暴露。

### 4.2 降级要可解释

如果返回 mock 或缓存结果，必须明确告诉上层为什么降级。

### 4.3 质量检查要轻量

质量检查只做必要的结构、空值、字段和时间顺序校验，不在这一层做复杂统计判断。

### 4.4 上层统一消费

分析器、回测、智能体工具尽量消费同一种快照结构，而不是各自拼接返回值。

---

## 5. 核心设计

### 5.1 数据快照

一次数据访问都应能够生成统一快照，建议字段如下：

- `name`
- `source`
- `data_source`
- `fetched_at`
- `is_degraded`
- `cache_hit`
- `reason`
- `quality`
- `meta`
- `payload`

### 5.2 质量检查

建议支持以下检查：

- DataFrame 是否为空
- OHLCV 字段是否完整
- 行数是否为零
- 索引是否可解析为时间
- 字典是否为空

### 5.3 降级策略

| 场景 | 策略 |
|------|------|
| 真实行情请求失败 | 使用本地模拟行情 |
| akshare 基本面失败 | 返回空值快照或默认指标 |
| 数据结构异常 | 返回降级快照并记录原因 |
| 缓存命中 | 直接返回缓存结果并标注 `cache_hit` |

## 6. 接口设计

### 6.1 行情

```python
def get_stock_hist(...): ...
def fetch_stock_hist_governed(...): ...
```

### 6.2 基本面

```python
class RealDataProvider:
    def get_financial_indicators(...): ...
    def get_financial_indicators_governed(...): ...
```

### 6.3 统一快照

```python
class DataSnapshot:
    def to_dict(self) -> dict: ...

class DataQualityChecker:
    def check_dataframe(self, df) -> dict: ...
    def check_ohlcv_dataframe(self, df) -> dict: ...
    def check_dict(self, data: dict) -> dict: ...
```

---

## 7. 关键验收标准

- 行情请求失败时能稳定回退到模拟数据
- 基本面请求失败时不抛出未处理异常
- 快照中能明确看到 `source` 和 `is_degraded`
- 回测结果能保留数据来源与降级原因
- 分析结果能保留数据治理信息
- 旧接口调用方式不被破坏

---

## 8. 归档要求

当以下内容完成后，可将本规格归档：

1. 行情和基本面至少一条真实数据路径接入治理层
2. 分析或回测入口能返回治理信息
3. 通过轻量验证确认降级路径可工作
4. 形成归档文档并沉淀到知识库
