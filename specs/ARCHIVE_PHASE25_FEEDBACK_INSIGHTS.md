# 归档文档：Phase 25 最小反馈洞察入口

---

## 1. 归档结论

Phase 25 已完成最小反馈洞察入口。

本阶段没有引入复杂分析平台，而是把现有决策会话、反馈记录和学习统计进一步收敛成只读洞察入口，用于回答“下一步最该优化什么”。

---

## 2. 已完成内容

- 新增 Phase 25 规格草案
- `DecisionSessionStore.get_feedback_insights()` 输出优化目标、高价值路径和样本提示
- `StockOrchestrator` 新增决策反馈洞察入口
- HTTP API 新增 `GET /decision/insights`
- `examples/api_demo.py` 支持查看洞察结果
- `README_QUANT.md` 补充洞察入口和 curl 示例

---

## 3. 验证目标

轻量验证的目标是确认洞察入口能够服务于真实迭代判断：

- 可以查看总反馈量
- 可以查看采纳率与平均评分
- 可以查看按意图、按工具、按场景的聚合结果
- 可以查看 `optimization_targets`
- 可以查看 `high_value_paths`
- 可以查看 `optimization_notes`

---

## 4. 当前收口边界

本阶段不做自动改路由、不做复杂 BI、不做新存储层。

目标只是把反馈数据变成更容易执行的优化判断，避免继续陷入局部无限优化。

---

## 5. 本阶段结论

反馈洞察入口的价值在于，它把项目从“能统计反馈”进一步变成“能指引下一步应该改哪里”。

这对私人私用、边测边改的场景尤其重要，因为它可以直接帮助你把精力放在最能推进项目的地方。

---

## 6. 后续建议

1. 先在真实使用里多看几次 `/decision/insights`
2. 结合洞察结果，只对高价值路径做小步优化
3. 后续若要扩展，优先优化统一输出、路由和模板，而不是增加更多分析层
