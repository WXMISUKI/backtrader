## 1. 背景

Phase 37 已经把日报、日报对比、周报模板、阶段报告模板和持仓语境收成一个复盘留档包。

但在验证中发现，留档包里仍然存在明显的结构套叠问题：

- 留档包标题会把周报和阶段报告的标题再次串起来
- 阶段报告会吞入周报标题，周报又吞入日报标题
- 同一个概念在不同层级中字段名不够统一

这会让留档包看起来像“层层拼接的中间产物”，而不是一个结构清楚的交付物。

因此，本阶段不增加新能力，只做留档包结构清理：

- 压平标题层级
- 统一字段命名
- 让留档包更像正式交付包

---

## 2. 目标

### 2.1 总目标

把复盘留档包的层级和命名整理平整，让它更适合正式保存和分享。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 标题压平 | 避免标题嵌套套娃 |
| 命名统一 | 同类字段使用稳定名字 |
| 结构清楚 | 留档包像一个完整交付物 |
| 兼容现有输入 | 不影响已存在的日报/周报/阶段报告 |
| 轻量改造 | 只做结构整理，不重算内容 |

## 4. 设计原则

### 4.1 标题不递归

留档包里的标题只表达当前层级，不引用上一层标题串接成长句。

### 4.2 字段稳定

同类字段尽量使用统一命名，例如 `title`、`summary`、`highlights`、`report_text`。

### 4.3 只整理，不重算

本阶段只清理现有结构，不增加新的分析计算。

### 4.4 保持可交付

结构整理后的留档包应当更适合保存、发送和后续归档。

---

## 5. 输入输出约定

### 5.1 输入

继续沿用 Phase 37 的统一入口输入，并保留：

- `--archive-package`
- `--archive-name`

### 5.2 输出

建议优化 `archive_package`，使其包含：

- `title`
- `summary`
- `sections`
- `report_text`
- `daily_report`
- `daily_comparison`
- `weekly_report`
- `stage_report`
- `portfolio_summary`

其中 `sections` 建议按固定顺序组织：

1. 复盘总览
2. 当日日报
3. 日报对比
4. 周报模板
5. 阶段报告
6. 持仓语境

---

## 6. 验收标准

- 留档包标题不再嵌套套娃
- 留档包字段命名统一
- 留档包结构更适合交付和保存
- 现有日报/周报/阶段报告不受影响
- JSON 输出更清晰，便于后续读取

## 8. 下一步建议

1. 先把留档包标题压平，不再从周报/阶段报告里递归拼接

3. 最后把结构清理后的留档包写进归档，作为正式交付版本
