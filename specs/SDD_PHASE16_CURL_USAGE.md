## 1. 背景

Phase 12 到 Phase 15 已经补齐了最小 HTTP API、CLI、联调用例和运行说明。

为了让常用调试方式更直接，本阶段只补最小 `curl` 使用示例，方便快速验证接口。

---

## 2. 目标

### 2.1 总目标

提供一组可直接复制执行的 `curl` 示例，覆盖健康检查、统一决策和反馈提交。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 易复制 | 可直接复制到终端执行 |
| 易调试 | 覆盖核心三个接口 |
| 少依赖 | 不依赖额外客户端 |
| 结果清晰 | 请求方式和 JSON 结构明确 |

---

## 3. 示例范围

建议覆盖：

1. `GET /health`
2. `POST /decision`
3. `POST /feedback`

---

## 4. 设计原则

### 4.1 最小且常用

只保留最常用、最容易复制的示例。

### 4.2 直接可运行

示例应尽量不依赖复杂参数拼装。

### 4.3 保持一致

示例字段应与 API 实际输入输出保持一致。

---

## 5. 验收标准

- 可以用 `curl` 调用 `/health`
- 可以用 `curl` 调用 `/decision`
- 可以用 `curl` 调用 `/feedback`
