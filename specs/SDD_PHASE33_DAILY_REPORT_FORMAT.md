## 1. 背景

Phase 31 已经提供了统一入口，Phase 32 又在统一入口上增加了更短的日更摘要。

但当前输出仍然更偏“控制台结果”，适合快速查看，却不够像一份稳定的日更报告。日常留档和回看时，用户往往希望看到的是：

- 一段当天总览
- 数据健康结论
- 持仓语境
- 今日重点和风险
- 可留档的摘要文本

所以本阶段不再新增判断，而是把 Phase 32 的日更摘要整理成更稳定的报告格式。

---

## 2. 目标

### 2.1 总目标

把统一入口的日更摘要收口成一份更适合留档、复制和回看的日更报告。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 更像报告 | 输出结构更稳定、语义更完整 |
| 便于留档 | 报告文本可以直接保存或复制 |
| 保持轻量 | 不增加新的判断引擎 |
| 兼容控制台 | 仍然保留完整明细输出 |
| 支持 JSON | 报告文本和结构化字段都可导出 |

## 4. 设计原则

### 4.1 报告优先

把摘要整理成自然语言报告，而不是只是一串控制台标签。

### 4.2 结构稳定

报告段落顺序固定，便于长期阅读和对比。

### 4.3 复用已有结果

报告只整合 Phase 31 / 32 已有结果，不重新计算新指标。

### 4.4 可留档

报告文本可以直接写入 JSON，后续可复盘、归档或复制到外部文档。

---

## 5. 输入输出约定

### 5.1 输入

继续使用 Phase 31 的统一入口输入：

- `--watchlist`
- `--portfolio`
- `--start-date`
- `--end-date`
- `--risk-profile`
- `--limit`
- `--output-json`

### 5.2 输出

建议新增一个 `daily_report` 对象，包含：

- `headline`
- `overview`
- `health_summary`
- `portfolio_summary`
- `watchlist_summary`
- `highlights`
- `report_text`

报告文本建议包含：

- 今日总览
- 数据健康
- 持仓语境
- 今日重点
- 今日风险

---

## 6. 验收标准

- 统一入口能生成结构化日更报告
- 报告文本适合直接留档
- 控制台仍保留完整输出
- JSON 中保留报告结构和文本
- 不破坏 Phase 31 / 32 的现有行为
