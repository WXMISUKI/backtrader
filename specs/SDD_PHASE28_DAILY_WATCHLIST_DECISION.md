## 1. 背景

当前项目已经具备：

- 个股分析能力
- 统一决策入口
- 统一四段式摘要
- 反馈统计与洞察

但从“私人私用、快速投产、真实测试、持续迭代”的定位出发，单次分析还不够直接。更有价值的形态是：每天拿一份自选股清单，快速看出哪些值得继续关注，哪些暂不行动，哪些因为数据不足需要谨慎。

本阶段不追求工作台、前端或复杂调度，而是把现有能力收成一个可每天执行的最小清单。

---

## 2. 目标

### 2.1 总目标

构建一个“自选股日常决策清单”MVP，让用户可以用本地配置快速批量分析自选股，并得到可直接阅读的分组结果。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 直接可用 | 每天可以运行一次，快速得到清单结果 |
| 低成本 | 复用现有个股分析和编排能力，不引入复杂新系统 |
| 可读性高 | 每只股票输出统一摘要，方便人读和后续反馈 |
| 支持分组 | 自动把结果分成重点关注、继续观察、暂不行动、数据不足 |
| 可导出 | 允许把结果保存为 JSON，便于复盘和留档 |

## 4. 设计原则

### 4.1 复用现有能力

优先复用 `StockOrchestrator.analyze()` 和现有分析结果，不重复实现个股分析逻辑。

### 4.2 先分组，再解释

先给出可执行的分组，再给出每只股票的四段式摘要，降低阅读成本。

### 4.3 只做轻量状态判断

分组规则以推荐动作、置信度和数据治理状态为主，不引入复杂评分系统。

### 4.4 允许数据降级

如果结果来自 mock、缓存或明显降级数据，要明确提示，不把它伪装成高可信结果。

## 5. 输入输出约定

### 5.1 配置输入

建议使用 `config/watchlist.json`，每条记录至少包含：

- `stock_code`
- `name`

可选字段：

- `risk_profile`
- `tags`
- `reason`
- `enabled`

### 5.2 脚本输入

建议支持：

- `--watchlist`
- `--risk-profile`
- `--output-json`
- `--limit`

### 5.3 输出结构

输出至少应包含：

- 总体统计
- 分组结果
- 每只股票的四段式摘要
- 原始分析结果的最小摘要信息

每只股票建议输出：

- `结论`
- `依据`
- `风险`
- `下一步动作`
- `action`
- `confidence`
- `data_source`

---

## 6. 分组规则

### 6.1 重点关注

满足以下条件时进入重点关注：

- `action = BUY`
- `confidence >= 0.65`
- 数据不处于明显降级状态

### 6.2 继续观察

满足以下条件时进入继续观察：

- `action = HOLD`
- 或 `BUY` 但置信度未达到重点关注阈值

### 6.3 暂不行动

满足以下条件时进入暂不行动：

- `action = SELL`
- 或置信度较低且不适合继续加关注

### 6.4 数据不足

满足以下条件时进入数据不足：

- `data_source = mock`
- 或治理信息提示结果降级
- 或分析失败

---

## 7. 验收标准

- 可以从本地 watchlist 配置读取股票列表
- 可以批量输出自选股日常决策清单
- 每只股票都有统一的四段式摘要
- 结果能自动分组
- 可以选择性导出 JSON
- 不破坏现有 `decision`、`feedback`、`stats`、`insights` 链路
