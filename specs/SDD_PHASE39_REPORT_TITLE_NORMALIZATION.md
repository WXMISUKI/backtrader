## 1. 背景

Phase 37 和 Phase 38 已经把复盘留档包打包出来，并对结构做了初步压平。

但在实际输出中，仍然可以看到少量标题回声和摘要重复问题，例如：

- 周报标题仍可能带着日报的口吻
- 阶段报告标题仍可能包含周报的上下文
- 留档包的总览可能把多个层级的标题串在一起

这会让最终交付物看起来仍有一点“中间产物”的痕迹。

本阶段不增加任何新报告层，只做标题和摘要归一化：

- 各层只保留自己的标题语义
- 各层摘要只表达本层内容
- 留档包总览只做一层汇总，不再串联上层标题

---

## 2. 目标

### 2.1 总目标

把所有报告层的标题与摘要统一成一套更稳定、更扁平的命名和表达方式。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 去回声 | 去掉标题和摘要的套叠回声 |
| 命名统一 | 各层使用稳定字段和表达 |
| 交付清晰 | 留档包更像正式交付物 |
| 保持轻量 | 只改表达，不改判断 |
| 兼容现有输入 | 不影响现有使用方式 |

## 4. 设计原则

### 4.1 标题只属于本层

每层标题只描述当前层，不回头引用更低层标题。

### 4.2 摘要只表达当前层

每层摘要只保留当前层信息，避免串接多层语义。

### 4.3 留档包总览单层化

留档包总览只汇总关键结论，不重复堆叠标题。

### 4.4 不改判断，只改表达

本阶段只统一命名和文本表达，不调整逻辑和分组规则。

---

## 5. 输入输出约定

### 5.1 输入

继续使用 Phase 38 的统一入口输入，并保留：

- `--archive-package`
- `--archive-name`
- `--weekly-reports`
- `--stage-reports`
- `--compare-with`

### 5.2 输出

建议统一以下字段表达：

- `title`
- `summary`
- `sections`
- `report_text`

各层建议规范为：

- 日报：`日报总览`
- 周报：`周报总览`
- 阶段报告：`阶段总览`
- 留档包：`复盘总览`

---

## 6. 验收标准

- 周报标题不再引用日报标题
- 阶段报告标题不再引用周报标题
- 留档包总览不再串联多层标题
- 各层摘要表达更平整
- JSON 输出更容易直接阅读

## 8. 下一步建议

1. 先把周报、阶段报告和留档包的标题统一到同一套表达
