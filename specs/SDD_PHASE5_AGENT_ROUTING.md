## 1. 背景

当前智能体客户端已经能够调用项目工具，但意图路由仍主要依赖关键词匹配。

这种方式在简单问题上可用，但当用户问题更接近自然语言、或同时包含多个意图时，容易出现：

- 路由优先级不稳定
- 工具选择不准确
- 对长线/短线/风险/回测等复合意图解析不一致
- 上层难以解释为什么选了这个工具

因此需要一层更明确的意图路由规则。

---

## 2. 目标

### 2.1 总目标

把智能体的工具路由从“粗关键词猜测”升级为“可解释、可扩展、可统一维护”的意图路由。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 路由稳定 | 相似问题尽量得到一致工具选择 |
| 路由可解释 | 能说清楚为什么选这个工具 |
| 路由可扩展 | 后续加新意图时不会破坏旧逻辑 |
| 路由一致 | `ArkAgentClient` 与 `StockOrchestrator` 语义尽量一致 |

---

## 3. 范围

### 3.1 纳入范围

- 个股分析
- 推荐
- 市场概览
- 风控
- 回测
- 报告
- 基本面

### 3.2 不纳入范围

- 模型微调
- 意图分类器训练
- 多轮对话记忆

---

## 4. 设计原则

### 4.1 优先业务意图

优先匹配“分析 / 推荐 / 市场 / 风控 / 回测 / 报告 / 基本面”等业务意图。

### 4.2 复合意图有明确优先级

如果用户同时提到多个意图，按预定义优先级决定主路由，避免模型自由发挥。

### 4.3 结果可解释

路由结果应包含：

- 主意图
- 命中规则
- 候选工具
- 默认参数

---

## 5. 路由规则

### 5.1 优先级建议

1. 回测
2. 市场概览
3. 风控
4. 报告
5. 推荐
6. 基本面
7. 个股分析

### 5.2 默认路由

- 未识别到明确意图时，默认推荐
- 未指定风险偏好时，默认 `moderate`

### 5.3 结果结构

```json
{
  "intent": "recommend",
  "tool": "recommend_by_risk",
  "confidence": 0.9,
  "reason": "命中推荐类关键词",
  "arguments": {
    "risk_profile": "moderate",
    "top_n": 5
  }
}
```

---

## 6. 验收标准

- 关键意图都能稳定命中
- 结果中可解释路由原因
- `ArkAgentClient` 和 `StockOrchestrator` 可以逐步收敛到相似规则
- 新增工具时不破坏旧路由

---

## 7. 归档要求

当以下内容完成后，可将本规格归档：

1. 路由规则实现完成
2. 至少一个入口支持结构化路由结果
3. 轻量验证通过
4. 形成归档文档
