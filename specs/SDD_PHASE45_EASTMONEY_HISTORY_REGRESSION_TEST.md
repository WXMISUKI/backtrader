## 1. 背景

Phase 44 已经把东方财富历史行情失败诊断补到了 `failure_kind` 和 `fallback_reason`。

但如果没有一个稳定的回归测试，这个诊断层以后很容易被改回去，重新退化成只有 `mock`、没有原因的状态。

因此，本阶段不再增加诊断字段，而是把这层诊断用最小测试固定下来。

---

## 2. 目标

### 2.1 总目标

为东方财富历史行情失败诊断建立一个最小、稳定、可重复的回归测试。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 固定失败语义 | 远端断开时应标记为 request |
| 固定回退结果 | 失败后仍会回退到 mock |
| 固定诊断输出 | `failure_kind` 和 `fallback_reason` 应稳定存在 |
| 防止回退退化 | 后续改动不能把诊断字段改没 |

---

## 3. 范围

### 3.1 纳入范围

- 新增最小回归测试
- 验证 `fetch_stock_hist_governed`
- 更新归档与任务清单

### 3.2 不纳入范围

- 新增更多接口测试
- 新增复杂网络模拟
- 重写数据获取逻辑
- 新增代理或重试机制

---

## 4. 设计原则

### 4.1 最小可复现

只覆盖当前已经确认的问题类型，不把测试扩散到所有东财场景。

### 4.2 先锁语义

先锁住 `failure_kind` 和 `fallback_reason` 的语义，再考虑别的诊断增强。

### 4.3 保持轻量

测试应该是快速、确定性的，不依赖真实网络。

---

## 5. 输入输出约定

### 5.1 输入

测试建议通过 monkeypatch 模拟：

- `get_stock_hist` 抛出连接中断异常
- `DataQualityChecker.check_ohlcv_dataframe` 返回正常质量结果

### 5.2 输出

测试应确认：

- `data_source == "mock"`
- `meta.failure_kind == "request"`
- `meta.fallback_reason` 非空
- `quality.ok == true`

---

## 6. 验收标准

- 能稳定复现失败诊断语义
- 不依赖真实网络
- 能防止未来回归到无诊断的 mock 回退
- 不影响现有成功路径
