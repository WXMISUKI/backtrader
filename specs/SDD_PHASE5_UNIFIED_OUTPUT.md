## 1. 背景

当前项目的市场概览、风控、回测和推荐能力已经存在，但输出结构仍不完全一致。

如果智能体在调用这些工具时，每个工具返回的字段、来源和降级语义不一致，就会导致：

- 路由难以稳定
- 多轮工具调用后难以总结
- 审计和复验难以统一
- 用户看到的解释口径不一致

因此需要将这几类能力统一为一套输出契约。

---

## 2. 目标

### 2.1 总目标

让市场概览、风控和回测输出与分析、推荐保持同一治理语义和统一输出结构。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 输出一致 | 统一 `ok / tool / category / data_source / summary / data / meta` |
| 治理一致 | 统一 `quality / governance / is_degraded` 语义 |
| 路由一致 | 智能体可按同一 schema 理解不同工具 |
| 归档一致 | 每个工具都有明确的已完成状态和后续计划 |

---

## 3. 范围

### 3.1 纳入范围

- `get_market_overview`
- `get_risk_profile`
- `run_backtest`
- `StockOrchestrator`
- 智能体工具注册表

### 3.2 不纳入范围

- 新市场数据源接入
- 新策略引擎
- 实盘交易
- 复杂权限系统

---

## 4. 设计原则

### 4.1 业务级输出优先

工具要面向“分析结论”和“治理信息”输出，而不是把原始对象直接扔给上层。

### 4.2 统一快照优先

所有结果尽可能包含：

- 数据来源
- 降级状态
- 质量结果
- 元数据

### 4.3 兼容旧接口

老的调用方式保持不变，新增字段只做增强，不做破坏。

---

## 5. 核心设计

### 5.1 市场概览

市场概览返回结果应包含：

- 指数
- 热门板块
- 市场情绪
- 趋势判断
- 治理信息

### 5.2 风控

风控返回结果应包含：

- 风险等级
- 止损止盈
- 最大仓位
- 风险说明
- 治理信息

### 5.3 回测

回测返回结果应包含：

- 收益摘要
- 风险摘要
- 交易统计
- 数据来源
- 是否降级
- 数据质量

---

## 6. 验收标准

- 三类能力都能返回统一工具信封
- `data_source` 在结果中可见
- `governance` 在结果中可见
- 智能体工具注册表可以直接消费
- 缓存命中时不破坏输出结构

---

## 7. 归档要求

当以下内容完成后，可将本规格归档：

1. 工具层已经统一输出
2. 编排层能稳定聚合这些结果
3. 轻量验证通过
4. 完成归档文档
