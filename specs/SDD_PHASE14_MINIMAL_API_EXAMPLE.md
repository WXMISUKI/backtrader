## 1. 背景

Phase 12 和 Phase 13 已经提供了最小 HTTP API 与 CLI 启动方式。

为了让这个闭环更容易验证和交付，本阶段补一个最小联调用例，用来快速检查：

- 健康检查是否正常
- 统一决策入口是否正常
- 反馈提交是否正常

---

## 2. 目标

### 2.1 总目标

提供一个可直接运行的最小 API 联调用例，帮助开发和联调。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 易运行 | 一条命令即可执行示例 |
| 易联调 | 能覆盖健康检查、决策和反馈 |
| 少依赖 | 仅使用标准库 |
| 结果清晰 | 输出 JSON 结果，便于排查 |

---

## 3. 示例流程

建议示例按以下顺序执行：

1. 请求 `GET /health`
2. 请求 `POST /decision`
3. 如果具备必要字段，再请求 `POST /feedback`

---

## 4. 设计原则

### 4.1 不新增依赖

示例只使用 Python 标准库，避免引入额外环境成本。

### 4.2 不替代业务逻辑

示例只负责联调验证，不承担核心业务编排。

### 4.3 输出可读

示例输出尽量保持结构化，便于快速确认服务状态。

---

## 5. 验收标准

- 可以运行联调用例
- 可以成功访问 `/health`
- 可以成功访问 `/decision`
- 如启用反馈步骤，可以成功访问 `/feedback`
