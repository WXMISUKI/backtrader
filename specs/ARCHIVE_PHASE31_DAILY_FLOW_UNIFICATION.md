# 归档文档：Phase 31 自选股日常流程统一入口

---

## 1. 归档结论

Phase 31 已完成自选股日常流程统一入口。

本阶段没有再增加分析能力，而是把 Phase 28、29、30 分别做成的三块能力收成一条统一运行路径，让日常使用从“先挑脚本”变成“直接跑一个命令”。

---

## 2. 已完成内容

- 新增 Phase 31 规格文档
- 新增 `examples/watchlist_shared.py`，统一承载 watchlist / portfolio 加载和持仓上下文构建
- 新增 `examples/daily_watchlist_pipeline.py`
- 统一入口按顺序输出：
  - 数据健康预检
  - 持仓上下文
  - 日常决策清单
- 统一入口支持 JSON 导出
- `README_QUANT.md` 补充统一入口运行方式
- `specs/TASK_CHECKLIST.md` 更新 Phase 31 状态

---

## 3. 验证目标

轻量验证的目标是确认统一入口能稳定串起三个阶段的能力：

- 可以读取 watchlist
- 可以读取 portfolio
- 可以执行数据健康预检
- 可以执行日常决策分析
- 可以输出统一 JSON

---

## 4. 当前收口边界

本阶段只做统一入口，不删除旧脚本，不重写已有分析逻辑，也不增加新的交易能力。

这条链路的职责是把用户每天要做的三件事收拢成一件事，不是把项目改成另一个系统。

---

## 5. 本阶段结论

统一入口的价值在于减少日常操作成本和认知切换成本。

这比继续抠单个字段更有投产价值，因为它直接把“能用”推进到了“顺手用”。

---

## 6. 后续建议

1. 先让统一入口连续跑几天，观察输出是否足够稳定
2. 如果确实好用，再考虑把旧脚本的说明进一步弱化，统一入口设为默认推荐
3. 如果后续还要继续深挖，可以把运行结果再串成更简洁的日更摘要
