## 1. 背景

Phase 12 已有最小 HTTP API，Phase 13 已有 CLI 启动方式，Phase 14 已有最小联调用例。

但当前仍缺少一份直接面向使用者的最小运行说明，导致“能跑”还没有完全变成“可交付”。

本阶段只做最小运行手册，不扩展业务、不增加新的运行框架。

---

## 2. 目标

### 2.1 总目标

提供一份可直接执行的最小运行说明，覆盖启动、联调和最基本的调用路径。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 可启动 | 说明如何启动最小 HTTP API |
| 可联调 | 说明如何使用联调用例 |
| 可复用 | 说明默认参数和环境变量 |
| 可交付 | 说明最小使用步骤 |

---

## 3. 内容范围

建议说明至少覆盖：

1. conda 环境激活
2. 启动 API 服务
3. 使用联调用例
4. 可选提交反馈
5. 常用环境变量

---

## 4. 设计原则

### 4.1 只写最小步骤

只保留当前项目真正可执行的路径，避免写成过度完整的运维手册。

### 4.2 以默认路径为主

优先展示本地默认启动方式，再说明可选配置。

### 4.3 直达联调

说明应帮助用户快速验证 `/health`、`/decision`、`/feedback`。

---

## 5. 验收标准

- 可以按说明启动 API 服务
- 可以按说明运行联调用例
- 可以按说明完成最小闭环验证

---

## 6. 归档要求

当以下内容完成后，可将本规格归档：

1. README 或使用说明已更新
2. 轻量验证通过
3. 形成归档文档
