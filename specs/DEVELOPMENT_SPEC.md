# 股票量化交易智能体开发规格书

## 项目概述

### 项目名称
StockQuantAdvisor - 股票量化交易智能顾问系统

### 项目目标
构建一个基于 Skill 架构的股票量化交易智能体系统，提供个股分析、买卖建议、风险控制等功能。

### 技术栈
- **回测引擎**: backtrader (已有)
- **数据源**: akshare + 东方财富API (已有)
- **技术指标**: TA-Lib / pandas-ta
- **AI框架**: Claude Skill

## 开发注意事项与规范

### 1. 先看规范，再动代码

- 新任务优先阅读 `specs/` 下的 SDD、路线图和任务清单，再开始改代码。
- 如果是新方向，先补规格，再实现，再归档，不跳步。
- 避免只改一个局部入口就结束，优先把同一条链路的展示、验收和归档一起补齐。

### 2. 以投产价值优先，不做局部无限优化

- 优先处理能直接推进项目投产、降低失败率、提高日常可见性的内容。
- 不把时间耗在无穷细的调参、改名和重复证明上。
- 数据源、路由、质量门禁、验收入口优先于体验型装饰。

### 3. 常用开发环境约定

- Python 相关验证优先切换项目对应的 conda 环境，不默认使用 `base`。
- 项目里常见环境：
  - `quant`：主开发和回归验证优先环境
  - `akshare-bt`：专门用于 AKShare 相关联调和兼容性检查
- 如果某个环境缺少依赖，先补依赖再测试，不要用错误环境凑结果。

### 4. 常用工具约定

- 代码修改优先使用 `apply_patch`。
- 文件搜索优先用 `rg` / `rg --files`。
- 结构化理解优先用 CodeGraph，必要时先 `sync` 再查 `status`。
- 做完文档或代码更新后，先确认索引状态，再结束阶段。

### 5. 测试与验证约定

- 默认只做轻量验证，不强制跑完整构建。
- 除非改动很重要，否则不执行 `npm build`。
- 新增的临时测试文件在验证通过后要删除，避免把一次性验证留在仓库里。
- 现有测试只在相关范围内补充，不为了验证而无限扩测。

### 6. 文档与归档约定

- 每个阶段按 `规格 -> 实现 -> 归档` 的顺序闭环。
- 代码改动后，同步更新 `README_QUANT.md`、路线图和任务清单。
- 归档文档只记录阶段结论、完成项、验收结果和后续建议，避免把正文写成操作日志。

### 7. 历史行情数据源约定

- 历史行情默认按多源思路理解，不再把单一数据源当成唯一入口。
- 常见顺序是：东方财富 -> AKShare -> 离线兜底。
- 健康摘要和流水线输出里要能看见 `selected_provider`、`provider_attempts` 和相关失败语义。

### 8. 质量门禁协作约定

- 统一质量门禁不是单一评分规则，而是智能体、提示词、Skill、质量摘要和验收入口协作出来的一套判断。
- 门禁语义要和 `production_gate`、`action_list`、`prompt_context`、`review_brief`、`schedule_hint` 保持一致。
- 如果质量明显降级，优先让输出可解释，而不是硬给强行动建议。

### Skill 交互流程

```
用户输入: "分析一下平安银行能不能买"
        ↓
[stock-advisor skill]
        ↓
┌─────────────────────────────────────┐
│  1. 获取最新行情数据                 │
│  2. 计算技术指标 (MA/MACD/RSI/KDJ)  │
│  3. 分析当前趋势状态                 │
│  4. 生成买卖信号                     │
│  5. 评估风险等级                     │
│  6. 输出结构化建议                   │
└─────────────────────────────────────┘
        ↓
返回: 买入/观望/卖出 + 目标价 + 止损价 + 置信度
```

## 买卖信号规则

### 买入信号条件 (满足2个以上)

1. **均线金叉**: MA5 上穿 MA20
2. **MACD金叉**: DIF 上穿 DEA
3. **RSI超卖反弹**: RSI < 30 后回升
4. **KDJ金叉**: K线上穿D线
5. **放量突破**: 成交量 > 5日均量 * 1.5

### 卖出信号条件 (满足2个以上)

1. **均线死叉**: MA5 下穿 MA20
2. **MACD死叉**: DIF 下穿 DEA
3. **RSI超买**: RSI > 70
4. **KDJ死叉**: K线下穿D线
5. **跌破止损**: 亏损超过止损阈值

### 信号强度计算

```
信号强度 = Σ(指标权重 × 信号值) / 总权重

买入强度 > 0.6 → 强烈买入
买入强度 0.4-0.6 → 建议买入
买入强度 0.2-0.4 → 观望
买入强度 < 0.2 → 不建议买入
```
