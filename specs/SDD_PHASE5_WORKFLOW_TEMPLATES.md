## 1. 背景

工作流执行器已经具备：

- 协作规划
- 工作流执行
- 生命周期审计
- 节点级 fallback

但如果每次复杂请求都临时生成计划，工作流会过于依赖输入文本质量，导致：

- 结果不稳定
- 节点顺序漂移
- 审计可比性差
- 后续优化难复用

因此需要把高价值路径收敛成标准工作流模板。

---

## 2. 目标

### 2.1 总目标

将高频高价值的协作链路固化成标准工作流模板，让智能体能稳定复用。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 模板化 | 常见场景使用固定模板 |
| 稳定性 | 任务顺序和依赖固定 |
| 可解释 | 模板名称和选择原因可见 |
| 可扩展 | 后续新增模板不影响旧模板 |
| 可审计 | 模板选择和执行都可回放 |

---

## 3. 推荐模板

### 3.1 市场风控分析模板

适用场景：

- 市场概览 + 风控 + 个股分析 + 报告
- 用户先问大盘，再问某只股票是否值得看

建议链路：

1. `get_market_overview`
2. `get_risk_profile`
3. `analyze_stock`
4. `generate_stock_report`

### 3.2 推荐验证模板

适用场景：

- 推荐 + 回测 + 报告
- 用户想知道“推荐是否值得执行”

建议链路：

1. `get_market_overview`
2. `recommend_by_risk`
3. `run_backtest`
4. `generate_stock_report`

### 3.3 选股研究模板

适用场景：

- 选股 + 基本面 + 风控 + 报告
- 用户希望先筛候选再深入分析

建议链路：

1. `screen_stocks`
2. `analyze_fundamental`
3. `get_risk_profile`
4. `generate_stock_report`

## 4. 模板选择规则

### 4.1 优先级建议

1. 显式工作流请求
2. 推荐验证模板
3. 市场风控分析模板
4. 选股研究模板
5. 默认通用模板

### 4.2 触发条件

- 用户提到 `工作流`、`流程`、`串联`、`联动` 时，优先进入模板选择
- 用户提到 `推荐`、`回测`、`验证` 时，优先选择推荐验证模板
- 用户提到 `市场`、`风控`、`分析`、`报告` 时，优先选择市场风控分析模板
- 用户提到 `选股`、`筛选`、`基本面` 时，优先选择选股研究模板

### 4.3 选择结果字段

建议模板选择结果包含：

- `template_id`
- `template_name`
- `template_reason`
- `selected_by`
- `tasks`
- `execution_order`
- `fallback`

### 4.4 模板工具暴露

建议同时提供一个轻量的模板查看入口，供智能体和调试工具直接查看当前可复用模板：

- `list_workflow_templates`

该入口不参与业务执行，只负责返回模板目录，便于：

- 智能体在路由前做模板理解
- 调试人员快速确认当前模板集合
- 后续新增模板时保持可发现性

---

## 5. 工作流输出要求

模板执行结果应在现有工作流 schema 上增加：

- `template_id`
- `template_name`
- `template_reason`
- `template_hit`

同时，模板命中信息应进入工作流审计和结果元数据，便于后续回放和优化。

---

## 6. 验收标准

- 常见高价值路径可以映射到稳定模板
- 模板输出的步骤顺序稳定可复用
- 模板选择可审计
- 新增模板不会破坏已有模板
- 现有非模板工作流仍可正常执行
