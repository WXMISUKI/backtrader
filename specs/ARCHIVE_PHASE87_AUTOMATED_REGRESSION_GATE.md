# Phase 87 归档：自动回归门禁与一键验证闭环

## 1. 背景

Phase 86 已经把 `quant` 环境收成了执行门禁，但日常验证仍然分散在多个入口里，缺少一个更薄的一键回归闭环。

Phase 87 的目标是把 `daily_run -> acceptance -> baseline` 收成一个自动回归门禁，让结果可以一次跑完、一次判定、一次回写。

## 2. 本次完成内容

- 新增 `examples/daily_watchlist_regression_gate.py`
- 回归门禁按 `daily_run -> acceptance -> baseline` 顺序执行
- 回归结果分类为 `pass / warn / block`
- 回写 `run_status`，若 `latest.json` 已存在也同步回写
- 新增 Phase 87 规格与归档文档

## 3. 结果

这一阶段的核心变化是：

- 日常回归从多入口执行推进到一键门禁
- 验证结果开始回写到现有产物，而不是只停在终端
- 门禁层保持很薄，没有引入新的业务分析或调度平台

## 4. 验收口径

- 可以一键执行关键日常验证步骤
- 结果能够区分通过、告警和阻断
- 回归结果能进入 `run_status` / `latest.json`
- 失败时能清楚知道卡在哪一步

## 5. 后续建议

Phase 87 之后，可以继续观察回归门禁是否需要再接入更细的提示语境，但不建议先扩新编排层或新仪表盘。
