## 1. 背景

Phase 40 已经把自选股统一流程的结果稳定落到日常归档目录中，并生成了：

- `latest.json`
- `latest.md`
- 按日期目录的归档文件

但当前还缺一个固定的查看入口：

- 每天跑完以后，不方便快速定位最近一次结果
- 需要手动找目录或翻开 JSON
- `latest.md` 虽然有了，但还没有一个统一命令来直接读它

因此，本阶段不再扩展分析，也不再增加新的留档内容，而是补一个最轻的“查看入口”，让日常归档真正可读、可找、可回看。

---

## 2. 目标

### 2.1 总目标

为 Phase 40 的日常归档提供一个稳定、轻量、直接可用的查看入口。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 快速查看 | 直接读取最近一次归档结果 |
| 自动回退 | `latest` 不存在时可回退到最新日期目录 |
| 轻量展示 | 默认输出简洁摘要，不展开全部原文 |
| 可选深看 | 需要时可直接打印 Markdown 或 JSON |
| 保持单向 | 只读不改，不影响现有归档流程 |

---

## 3. 范围

### 3.1 纳入范围

- 新增日常留档查看脚本
- 读取 `latest.json`
- 读取 `latest.md`
- 读取最近一次日期目录归档
- 更新运行说明
- 更新任务清单

### 3.2 不纳入范围

- 新增图形界面
- 新增网页门户
- 新增数据库索引
- 修改归档写入逻辑
- 修改分析与报告生成逻辑

---

## 4. 设计原则

### 4.1 默认只读

查看入口只做读取和展示，不改动留档内容。

### 4.2 优先最近结果

默认优先查看 `latest`，让日常回看路径最短。

### 4.3 回退稳定

如果 `latest` 不存在，自动回退到最近日期目录中的归档文件。

### 4.4 输出克制

默认只展示足够的摘要信息，不把整份 JSON 直接刷满终端。

---

## 5. 输入输出约定

### 5.1 输入

继续沿用 Phase 40 的归档目录约定，并新增查看脚本参数：

- `--archive-dir`
- `--show-markdown`
- `--show-json`
- `--limit-lines`

### 5.2 输出

建议输出三类内容：

- 归档位置和来源
- 日更摘要与关键报告标题
- 可选的完整 Markdown 或 JSON

### 5.3 回退顺序

建议按以下顺序寻找目标文件：

1. `archive-dir/latest.json`
2. 最新日期目录下的 JSON 文件
3. `archive-dir/latest.md`
4. 最新日期目录下的 Markdown 文件

---

## 6. 验收标准

- 可以直接查看最近一次留档结果
- `latest` 缺失时有稳定回退路径
- 默认输出简洁，不强制展开全部内容
- 可选 Markdown/JSON 展示可用
- 不影响 Phase 40 写入逻辑
