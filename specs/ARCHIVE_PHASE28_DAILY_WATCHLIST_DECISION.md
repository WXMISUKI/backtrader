# 归档文档：Phase 28 自选股日常决策清单 MVP

---

## 1. 归档结论

Phase 28 已完成自选股日常决策清单 MVP。

本阶段没有新增 HTTP 接口或复杂工作台，而是把现有个股分析能力收束成一条每天可执行的最小路径：读取本地 watchlist，批量分析，按动作和可信度分组，输出可直接阅读的清单。

---

## 2. 已完成内容

- 新增 Phase 28 规格文档
- 新增 `config/watchlist.json` 作为本地自选股样例
- 新增 `examples/daily_watchlist_decision.py`
- 脚本复用 `StockOrchestrator.analyze()` 批量分析自选股
- 脚本自动分组为 `重点关注 / 继续观察 / 暂不行动 / 数据不足`
- 脚本支持可选 JSON 导出
- `README_QUANT.md` 增加日常清单的运行方式
- `specs/TASK_CHECKLIST.md` 记录 Phase 28 完成状态

---

## 3. 验证目标

轻量验证的目标是确认这条最小路径可以稳定运行：

- 可以读取本地 watchlist
- 可以批量调用现有分析能力
- 可以输出分组和四段式摘要
- 可以选择性导出 JSON
- 不影响既有 `/decision`、`/feedback`、`/decision/stats`、`/decision/insights` 链路

---

## 4. 当前收口边界

本阶段只做日常清单，不做自动交易、不做前端工作台、不做新的数据库，也不重新设计分析算法。

分组规则以推荐动作、置信度和数据治理状态为主，目标是先让结果可用、可读、可复盘。

---

## 5. 本阶段结论

自选股日常决策清单是当前最贴近项目定位的落点之一。

它把项目已有的分析、摘要、反馈能力串成了一条真实可用的日常链路，能更快让系统从“可调用”走向“真正在用”。

---

## 6. 后续建议

1. 先在真实自选股上跑几天，观察分组和阈值是否顺手
2. 如果后续要增强，再考虑加持仓输入或更细的关注标签
3. 等日常使用稳定后，再把这条清单进一步接到反馈闭环里
