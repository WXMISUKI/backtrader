# Phase 89 归档：日常使用测试规范化与最小对话入口说明

## 1. 背景

Phase 89 的目标不是新增分析能力，而是把项目的日常使用方式收成一份明确、可重复、低摩擦的使用协议，并把当前是否存在前端对话入口说明清楚。

## 2. 本次完成内容

- 新增 `examples/daily_watchlist_usage_guide.py`
- 在 `README_QUANT.md` 中补充日常使用测试建议
- 明确当前没有独立的前端对话 AI 页面
- 说明当前可用替代入口为 CLI、HTTP API 和日常脚本
- 新增 Phase 89 规格文档

## 3. 结果

这一阶段的核心变化是：

- 日常使用测试有了明确顺序
- 用户不再需要自己猜今天先看什么
- 当前前端对话入口现状被明确说明，不再容易产生误解
- `daily_watchlist_usage_guide.py` 可以作为最小使用入口直接运行

## 4. 验收口径

- 用户能按固定顺序完成日常使用测试
- README 能直接指导用户开始使用
- 用户能明确知道当前没有前端对话 AI 页面
- 用户能明确知道当前可用的替代入口
- 不引入新的前端框架或调度平台

## 5. 后续建议

Phase 89 之后，如果要继续推进交互入口，建议单独起一个新阶段去做最小 Web Chat UI，但不建议和当前日常使用协议混在一起继续扩张。
