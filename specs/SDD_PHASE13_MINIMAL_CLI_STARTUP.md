## 1. 背景

Phase 12 已经提供了最小 HTTP API，但如果只依赖模块导入或手工执行函数，仍然不够“可直接投产”。

为了让项目更快投入生产，本阶段只补一个最薄的 CLI 启动方式，不引入新的运行框架或复杂参数体系。

---

## 2. 目标

### 2.1 总目标

提供一个可以一条命令启动 HTTP API 的最小 CLI 入口。

### 2.2 具体目标

| 目标 | 说明 |
|------|------|
| 一条命令启动 | 直接启动 `run_agent_api_server()` |
| 参数最少 | 只保留 `host` 和 `port` |
| 默认可用 | 提供合理默认值 |
| 易安装 | 既支持脚本执行，也支持安装后调用 |
| 易运维 | 环境变量可覆盖默认值 |

---

## 3. 启动入口设计

### 3.1 命令入口

建议提供：

- `stock-agent-api`

### 3.2 参数

最小参数集：

- `--host`
- `--port`

### 3.3 默认值

- `host = 127.0.0.1`
- `port = 8000`

也允许通过环境变量覆盖默认值：

- `STOCK_AGENT_API_HOST`
- `STOCK_AGENT_API_PORT`

---

## 4. 设计原则

### 4.1 只做薄层

CLI 只负责参数解析和启动，不再引入额外业务逻辑。

### 4.2 保持稳定

启动方式应尽量和当前 HTTP API 的实现保持一致，避免引入第二套启动逻辑。

### 4.3 面向快速交付

优先满足可启动、可配置、可安装，不做过度抽象。

---

## 5. 验收标准

- 可以通过 CLI 启动 HTTP API
- `host` 和 `port` 可配置
- 默认值可用
- 编译和轻量验证通过
