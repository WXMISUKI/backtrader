## 1. 背景

Phase 60 已经把日常运行、诊断证据和验收结果收成统一 `production_gate`，解决了“今天结果能不能用”的总开关问题。

但如果门禁仍然只看粗粒度状态，系统在面对“真实但质量一般”“缓存回退”“局部字段缺失”“部分股票明显低可信”这类情况时，仍然只能给出偏粗的判断。

Phase 61 的目标，就是把“数据可信度”正式引入日常链路，让健康摘要、决策分组和门禁结果都能共享同一套可信度口径。

---

## 2. 总目标

构建一层轻量、统一、可解释的数据可信度增强机制，让系统能够：

- 为每只股票生成 `data_confidence`
- 为数据问题生成归一化原因码
- 为健康摘要增加可信度分布
- 让低可信样本自动进入 `数据不足` 或谨慎分组
- 让门禁能基于可信度做更稳定的 `warn / block` 判断

---

## 3. 核心原则

### 3.1 可信度不是单一分数

可信度不是简单的 `health_score` 变体，也不是一个黑盒模型分数。

它应该由以下协作要素共同构成：

- 数据源类型
- 是否降级
- 质量检查结果
- 原始失败原因
- 是否命中缓存
- 是否回退到 mock 或 error

### 3.2 先统一，再消费

先在共享层形成统一可信度，再由决策清单、诊断证据和门禁消费，避免各个脚本自己算一套。

### 3.3 保持轻量

Phase 61 不引入数据库、不引入模型训练、不引入新策略。它只负责把现有数据治理信息收敛成稳定字段。

---

## 4. 范围

### 4.1 纳入范围

- 新增统一数据可信度计算
- 新增归一化原因码
- 健康摘要中增加可信度字段
- 日常决策中增加可信度门槛
- 门禁中增加可信度依据
- 更新运行说明、任务清单和归档

### 4.2 不纳入范围

- 新数据源
- 新数据库
- 新前端
- 自动交易
- 模型训练

---

## 5. 设计要点

### 5.1 可信度字段

建议至少为每只股票输出：

- `data_confidence`
- `confidence_level`
- `confidence_breakdown`
- `normalized_reasons`

### 5.2 可信度分级

建议分为：

| 等级 | 含义 |
|---|---|
| `high` | 数据来源稳定，质量正常，可正常参考 |
| `medium` | 存在少量降级或缓存回退，谨慎参考 |
| `low` | 存在明显降级、错误或缺失，优先进入数据不足 |

### 5.3 归一化原因码

把原始错误信息统一成可统计的原因码，例如：

- `missing_stock_code`
- `dataframe_empty`
- `missing_columns`
- `financial_unavailable`
- `source_mock`
- `source_error`
- `quality_dict_empty`

---

## 6. 输出契约

### 6.1 健康摘要新增字段

```json
{
  "data_confidence": 0.82,
  "confidence_level": "high",
  "confidence_breakdown": {
    "history": {
      "score": 0.84,
      "level": "high"
    },
    "fundamental": {
      "score": 0.80,
      "level": "high"
    }
  },
  "normalized_reasons": [
    "source:real",
    "quality:ok"
  ]
}
```

### 6.2 日常摘要新增字段

建议增加：

- `confidence_counts`
- `average_confidence`
- `confidence_summary`

### 6.3 门禁新增依据

`production_gate.evidence` 建议包含：

- 平均可信度
- 低可信样本数
- 高可信样本数
- 可信度分布

---

## 7. 决策联动规则

### 7.1 低可信样本处理

当股票可信度偏低时：

- 不进入重点关注
- 优先进入数据不足
- 如果已有持仓，可保留观察，但必须提示谨慎

### 7.2 高可信样本处理

当股票可信度高且推荐动作明确时：

- 允许进入重点关注
- 允许继续观察
- 不额外抬高结论强度，只是减少误判

## 9. 验收标准

- 每只股票都有可信度字段
- 可信度分布能在日常摘要里看到
- 低可信样本会自动进入数据不足或谨慎分组
- 门禁会把低可信度作为判断依据
- 归一化原因码能稳定统计，不再依赖零散原始文本
