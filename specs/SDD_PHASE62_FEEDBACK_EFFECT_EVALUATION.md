## 1. 背景

Phase 47 到 Phase 61 已经把反馈记录、反馈洞察、日常门禁、数据可信度都铺好了。

但目前的反馈链路还停留在“记录了什么”和“最近有什么趋势”，还没有真正进入“这批建议后续表现如何”的效果评估。

Phase 62 的目标，就是把历史反馈和后续留档快照串起来，形成一个轻量的效果评估视图，为下一阶段的行动清单收口提供依据。

---

## 2. 总目标

建立一个最小可用的反馈效果评估能力，至少回答以下问题：

- 最近的反馈有没有可评估样本
- 建议在后续快照中的回看表现如何
- 重点关注/继续观察/暂不行动这些样本的后续回报大致怎样
- 哪些反馈样本应该纳入后续评估，哪些因为数据不足而排除

---

## 3. 核心原则

### 3.1 先做轻量回看

Phase 62 不引入复杂的收益归因，也不做模型训练。

它只做轻量评估：

- 基于已归档快照
- 基于反馈记录中的基线价格
- 基于后续 1/3/5 个快照窗口

### 3.2 先有基线，再谈效果

反馈记录必须先保存当时的决策上下文、价格、可信度、门禁状态，再进入效果评估。

### 3.3 保持解释性

评估结果必须能直接看懂，不用额外解码。

---

## 4. 范围

### 4.1 纳入范围

- 在反馈记录中补充基线信息
- 从历史留档快照中提取后续窗口表现
- 输出 T+1 / T+3 / T+5 的轻量回看结果
- 输出命中率、平均回报和样本分布
- 在回看入口中展示效果评估
- 更新运行说明、任务清单和归档

### 4.2 不纳入范围

- 新数据库
- 实盘级收益归因
- 交易信号训练
- 自动交易
- 前端图表

---

## 5. 评估窗口

### 5.1 当前约定

由于项目当前主要依赖已归档快照，本阶段的 `T+1 / T+3 / T+5` 采用轻量定义：

- `T+1`：反馈发生后遇到的第 1 个后续快照
- `T+3`：反馈发生后遇到的第 3 个后续快照
- `T+5`：反馈发生后遇到的第 5 个后续快照

这是一种面向日常投产的近似回看方式，先解决“有没有趋势、方向大不大、样本够不够”的问题。

后续如果留档跨度足够，再考虑升级为严格的交易日窗口。

## 7. 评估规则

### 7.1 基线字段

每条反馈记录建议至少包含：

- `reference_generated_at`
- `reference_group`
- `reference_action`
- `reference_confidence`
- `reference_data_confidence`
- `reference_price`
- `reference_gate_status`

### 7.2 命中定义

本阶段采用轻量命中定义：

- `BUY` 或 `重点关注`：后续收益大于等于 0 视为命中
- `SELL`：后续收益小于等于 0 视为命中
- 其他动作：只统计回报，不强行算命中
